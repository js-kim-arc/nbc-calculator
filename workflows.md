## [Epic] 클래스 없이 동작하는 절차적 계산기를 구현한다

### 에픽 설명
객체지향 개념 없이 순차적인 흐름만으로 계산기의 핵심 동작을 완성한다.
Scanner 입력 → 연산 판단 → 결과 출력의 흐름을 직접 손으로 구성하면서,
제어문과 반복문의 역할을 체득하는 것이 목적이다.

### 에픽 내부 메모
- 구조 설계보다 "입력 → 처리 → 출력"의 흐름 이해가 우선
- 예외 처리(0 나눗셈)를 직접 분기로 경험하게 하는 것이 핵심
- 반복은 처음부터 넣는 게 아니라 마지막에 감싸는 형태로 수정

---

### Story 1. Scanner로 두 정수와 연산 기호를 입력받는다

**설명**
사용자로부터 양의 정수 2개와 사칙연산 기호를 순서대로 입력받아 각각 적절한 타입의 변수에 저장한다.

**완료 기준**
- [ ] `Scanner`를 사용해 양의 정수 2개를 하나씩 입력받을 수 있다.
- [ ] 두 정수는 정수형 변수에 각각 저장된다.
- [ ] 사칙연산 기호(`+`, `-`, `*`, `/`)를 문자형(`char`)으로 입력받아 저장한다. (`charAt(0)` 활용)

---

### Story 2. 입력값으로 사칙연산을 수행하고 결과를 출력한다

**설명**
입력받은 두 정수와 연산 기호를 바탕으로 연산을 수행하고, 결과를 출력한다.
연산 오류가 발생할 수 있는 경우(0으로 나누기)는 별도 메시지로 안내한다.

**완료 기준**
- [ ] `if` 또는 `switch`를 사용해 연산 기호에 맞는 연산을 수행한다.
- [ ] 네 가지 연산(`+`, `-`, `*`, `/`) 모두 처리된다.
- [ ] 0으로 나눌 경우 연산을 수행하지 않고 안내 메시지를 출력한다.
    - 예: `"나눗셈 연산에서 분모(두번째 정수)에 0이 입력될 수 없습니다."`
- [ ] 연산 결과는 콘솔에 출력된다.

---

### Story 3. 반복문을 추가해 계속 연산할 수 있도록 수정한다

**설명**
기존 연산 흐름 전체를 반복문으로 감싸, 사용자가 원할 때까지 계속 계산할 수 있게 한다.

**완료 기준**
- [ ] 반복문(`for` 또는 `while`)으로 입력 → 연산 → 출력 흐름 전체를 반복할 수 있다.
- [ ] 매 연산 후 계속 여부를 사용자에게 묻는다.
- [ ] `exit`을 입력하면 반복이 종료된다.
- [ ] Story 2의 모든 연산 및 예외 처리 동작이 그대로 유지된다.

## [Epic] Calculator 클래스를 도입해 책임을 분리한다

### 에픽 설명
절차적으로 작성된 계산기 코드를 객체지향 구조로 전환한다.
연산 책임과 결과 저장 책임을 `Calculator` 클래스에 위임하고,
`App`은 입력과 흐름 제어만 담당하도록 역할을 분리한다.

### 에픽 내부 메모
- 클래스는 "묶는 그릇"이 아니라 "책임의 단위"임을 체험하는 것이 목적
- 컬렉션 필드는 단순 저장이 아니라 상태 유지의 개념으로 접근
- 캡슐화는 규칙이 아니라 외부 의존을 줄이는 이유로 이해

---

### Story 1. 사칙연산과 결과 저장을 담당하는 Calculator 클래스를 구현한다

**설명**
두 정수와 연산 기호를 받아 연산을 수행하고 결과를 반환하는 메서드,
그리고 결과를 누적 저장하는 컬렉션 필드를 가진 `Calculator` 클래스를 작성한다.

**완료 기준**
- [ ] `Calculator` 클래스에 연산 결과를 저장하는 컬렉션 타입 필드가 선언된다.
- [ ] `calculate(num1, num2, operator)` 형태의 메서드가 사칙연산을 수행하고 결과를 반환한다.
- [ ] 연산 결과는 메서드 내부에서 컬렉션 필드에 자동으로 저장된다.
- [ ] 0으로 나누는 경우 등 연산 오류를 내부에서 처리한다.

---

### Story 2. App 클래스가 Calculator를 활용하도록 수정하고, 캡슐화를 적용한다

**설명**
`App`의 `main` 메서드에서 `Calculator` 인스턴스를 생성해 연산을 위임하도록 수정한다.
동시에 컬렉션 필드를 `private`으로 제한하고, 결과 삭제 기능을 메서드로 제공한다.

**완료 기준**
- [ ] `App`은 직접 연산하지 않고 `Calculator` 인스턴스의 `calculate()` 메서드를 호출한다.
- [ ] 연산 결과는 `Calculator` 내부 컬렉션 필드에 저장되며, 외부에서 직접 접근할 수 없다. (`private`)
- [ ] `removeResult()` 메서드를 통해 저장된 결과를 삭제할 수 있다.
- [ ] 수정 이후에도 Step 1의 반복, 입력, 연산, 예외 처리 동작이 동일하게 동작한다.
